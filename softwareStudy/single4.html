System programming basic

배열과 포인터(심화)

포인터(심화)

-포인터 산술-

포인터에 덧셈을 할 수 있으며, 그 결과는 데이터 타입에 따라 달라짐

int* pi;         // pi = 400, pi + 1 = 404, pi + 2 = 408
char* pc;          // pc = 400, pc + 1 = 401, pc + 2 = 402
double* pd;       // pd = 400, pd + 1 = 408, pd + 2 = 41

단순히 숫자 1을 더하는 것이 아니라, 다음 데이터를 가리키는 것 으로 생각하자

배열의 이름은 배열 첫 번째 요소의 주소를 의미함
int arr[10];
배열의 이름 arr과 &arr[0]은 같은 의미
이때, arr+1 = &arr[0]+4 = &arr[1]

-증감연산자-

int num1, num2, *p1, *p2, rabbit[] = {10, 20, 30};
p1 = rabbit;
num1 = *(++p1);              //①
p2 = rabbit;
num2 = ++(*p2);             //②

①은 p1에 저장된 주소 값을 하나 증가시키고, 그 주소에 저장된 값을 사용하겠다는 의미
따라서 p1에 저장된 주소는 rabbit + 1 = &rabbit[0] + 4 = &rabbit[1]이 됨
주소에 저장된 값을 참조하면 num1에 20이 저장됨

②는 p2주소에 있는 값을 하나 증가시켜 사용하겠다는 의미
따라서 p2에 저장된 주소는 &rabbit[0]
주소에 저장된 값을 하나 증가시키면 10+1 = 11이 num2에 저장됨

간접 참조 연산자보다 증감연산자 우선순위가 더 높다 는 점에 주의할 것 (증감 > 간접 참조 > 덧셈)
Cf) *p++는 *(p++)를 의미하므로, ②번과 같은 의미로 사용하려면 반드시 괄호를 사용해야 함

-배열과 포인터의 차이점-

배열의 이름은 배열의 시작 주소를 가리키는 상수 포인터
상수는 L-value로 사용할 수 없음
즉, 등호의 왼쪽에 올 수 없음

포인터는 변수이므로 L-value로 사용할 수 있음

Ex)
int *p, arr[MAX];
arr = p;      // error
arr = 100;     // error
p = arr;      // ok
p = arr + 2;    // ok

-이중 포인터(double pointer)-

포인터 자체도 변수이며 메모리에 저장됨

2중 포인터는 포인터 변수를 가리키는 포인터를 의미함

이중 포인터가 쓰이는 이유:변수에 저장된 값을 바꾸는 예를 생각해보자
단순히 값 하나만 바꾸는 경우는 큰 의미가 없음
swap 함수를 여러 번 호출해서 배열의 값을 하나씩 바꿀 수도 있지만, 시간이 많이 걸림
이 경우 배열을 가리키는 포인터를 만들고, 그 포인터를 서로 교환하면 빠르게 해결 가능

함수 포인터(function pointer)

함수 포인터는 함수의 주소를 저장하는 포인터
함수도 변수와 마찬가지로 컴파일 및 링크 후 메모리 특정 번지에 할당됨
함수 코드나 문자열 리터럴은 실행 중 변경되지 않으므로 읽기 전용인 텍스트 세그먼트에 할당됨
텍스트 세그먼트의 코드도 메모리 특정 번지에 로드되며 실행 중 함수의 주소로 사용됨

함수에 대한 포인터를 선언하려면 함수 포인터가 가리킬 함수의 원형이 필요함
변수에 대한 포인터는 가리킬 변수의 데이터 타입과 같이 선언함
함수 포인터도 가리킬 함수의 원형과 같은 형태로 선언함

-함수 포인터 선언-
함수포인터는 가리키려는 함수의 원형과 동일하게 선언함
기본 형식은 return_type (*pointer_name)(argument_type_list);

Ex)
-함수 원형: int sum(int a, int b);
-함수 포인터는 int (*fp)(int, int);
-만일 *fp을 괄호로 묶지 않으면, 정수 포인터를 리턴하는 함수의 원형이 됨

-함수 포인터 초기화-

함수 포인터도 변수이므로 초기화하지 않으면 쓰레기 값이 들어있음
아직 가리키는 함수가 없을 경우 NULL 포인터로 초기화
Ex) int (*fp)(int, int) = NULL;

특정 함수로 초기화하려면 함수 이름 앞에 주소연산자를 붙임(함수 포인터를 초기화할 때, 함수의 인자는 생략함)
Ex) int (*fp)(int, int) = &sum;

함수의 이름은 함수의 시작 주소를 의미하므로, 주소연산자를 생략할 수 있음
Ex) int (*fp)(int, int) = sum;

NULL로 초기화한 후 함수 주소를 넣을 수도 있음
Ex) fp = sum;

-함수 포인터 사용-

함수 포인터도 포인터이므로 주소를 담고 있음
간접 참조 연산자를 이용해서 가리키는 함수를 호출할 수 있음
Ex) printf("%d", (*fp)(10, 20));

함수 포인터를 함수의 이름처럼 사용할 수 있음
Ex) printf("%d", fp(10, 20));

가리키는 함수가 없을 때는 NULL을 저장하므로, 사용 전 NULL test를 하는 것이 안전함
Ex) if(fp) printf("%d", fp(10, 20))

2차원 배열(Two dimensional array)

배열의 요소가 배열로 이루어진 배열

2차원 배열을 행렬(Matrix)라고도 부름

2차원 배열은 아래처럼 선언하고 초기화할 수 있으며, 두가지 방법 모두 동일한 의미를 가짐
Ex) int arr[2][3] = {{1, 2, 3},{4, 5, 6}};
    int arr[2][3] = {1, 2, 3, 4, 5, 6};
